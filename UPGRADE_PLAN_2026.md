# Kocpy 功能升级计划报告

**基于深度竞品分析**
**生成时间**: 2026-07-11

---

## 📊 研究概述

本次深度研究覆盖了 **6款主流DIT软件**，进行了 **102次子代理搜索**，验证了 **13条核心声明**，发现 **3条被推翻的营销声明**。

### 研究竞品
1. **Silverstack** (Pomfort) - 功能最全面的DIT管理工具
2. **ShotPut Pro** (Imaginyells) - 校验算法多样性领先
3. **YoYotta** - LTO磁带归档领域领先
4. **OffShoot** (Hedge) - ASC MHL合规简洁工具
5. **iconik** - 云端协作生态最完整
6. **Kocard** (悦流) - 软硬一体移动备份

---

## 🎯 核心发现

### 1. 行业标准趋势

**ASC MHL 已成准入门槛**
- Netflix 强推 ASC MHL 标准
- Silverstack、ShotPut Pro、OffShoot、YoYotta 均已支持
- **Kocpy 缺失此功能将严重限制市场接受度**

**多目标校验备份是基础能力**
- 所有专业工具都支持
- Kocpy 已具备（✅ 4个目的地 + 多种哈希算法）

### 2. 竞品功能分布

```
功能全面性:  Silverstack > ShotPut Pro > YoYotta > OffShoot > Kocard
校验算法:    ShotPut Pro (7种) > Silverstack > YoYotta > Kocpy (3种)
LTO归档:    YoYotta (LTO-5~10) > Silverstack > 其他
元数据管理:  Silverstack (100+字段) > YoYotta > 其他
工作流自动化: Silverstack (Lua API) > 其他
云端协作:    iconik (65+集成) > Kocard > 其他
项目管理:    Kocpy (✅) > Silverstack > 其他
NAS归档:    ❌ 竞品均未作为核心 = Kocpy的机会
```

## 🚀 功能升级路线图

### 优先级 1: 行业标准合规 (必须) ⭐⭐⭐⭐⭐

#### 1.1 ASC MHL 支持
**重要性**: 行业准入门槛，Netflix等平台要求
**当前状态**: ❌ 不支持
**目标**: 完整支持 ASC MHL V1 标准

**实现内容**:
- 生成 ASC MHL sidecar 文件
- 支持嵌入式 MHL 保存
- 备份完成后自动验证 MHL
- 支持后期再验证

**技术方案**:
```typescript
interface ASCMHLData {
  version: '1.0'
  hashAlgorithm: string
  files: Array<{
    path: string
    hash: string
    size: number
    timestamp: string
  }>
  metadata: {
    software: string
    operator: string
    createdAt: string
  }
}
```

**预计工作量**: 2-3 周
**收益**: 进入专业DIT市场，满足Netflix等平台要求

---

#### 1.2 更多校验算法支持
**重要性**: 高
**当前状态**: MD5, SHA-1, SHA-256 (3种)
**目标**: 支持 7 种以上，包括 xxHash

**新增算法**:
- ✅ xxHash64 (ShotPut Pro 默认，最快)
- ✅ XXHashH3-64
- ✅ XXHashH3-128
- ✅ C4

**技术方案**:
- 集成 xxHash 库
- 提供算法选择界面
- 保持向后兼容

**预计工作量**: 1 周
**收益**: 性能提升，兼容性增强

---

### 优先级 2: 元数据管理 (重要) ⭐⭐⭐⭐

#### 2.1 素材元数据提取与管理
**重要性**: 高
**当前状态**: ❌ 仅视频首帧
**目标**: 支持 50+ 元数据字段

**支持的元数据**:
- 摄影机信息 (型号、序列号、固件版本)
- 拍摄参数 (ISO、快门、光圈、白平衡)
- 时间码 (TC、TC User Bits)
- 镜头信息 (焦距、光圈、对焦距离)
- 色彩空间 (ACES、Rec.709、LogC)
- GPS/位置信息
- 自定义元数据

**技术方案**:
```typescript
interface ClipMetadata {
  // 摄影机信息
  camera: {
    manufacturer: string
    model: string
    serialNumber: string
    firmware: string
  }
  // 拍摄参数
  exposure: {
    iso: number
    shutter: string
    aperture: string
    whiteBalance: number
  }
  // 时间码
  timecode: {
    record: string
    camera: string
    userBits: string
  }
  // 更多...
}
```

**实现步骤**:
1. 集成 ExifTool/MediaInfo
2. 支持主流摄影机格式 (ARRI, RED, Sony, Blackmagic)
3. 创建元数据数据库
4. 提供搜索和筛选功能

**预计工作量**: 4-6 周
**收益**: 专业级元数据管理，提升工作效率

---

### 优先级 3: 转码功能 (重要) ⭐⭐⭐⭐

#### 3.1 智能代理文件生成
**重要性**: 高
**当前状态**: ❌ 无
**目标**: 生成编辑代理文件

**支持格式**:
- **输出格式**: ProRes Proxy, H.264, H.265/HEVC
- **分辨率**: 1080p, 720p, 480p
- **质量**: 可配置 (高/中/低)

**特色功能** (借鉴Kocard):
- 10-bit HEVC 转码 (文件体积减小80%)
- LUT/CDL 应用
- 批量转码
- 转码队列管理

**技术方案**:
- 集成 FFmpeg (已内置)
- 支持硬件加速 (Apple Silicon, Intel QSV)
- 进度监控和取消

**预计工作量**: 4-5 周
**收益**: 完整的 DIT 工作流，减少后期软件依赖

### 优先级 4: NAS 归档 (差异化) ⭐⭐⭐⭐⭐

#### 4.1 NAS 专项归档功能
**重要性**: 极高 (差异化机会)
**当前状态**: ⚠️ 基础支持
**目标**: 专业级 NAS 归档解决方案

**为什么这是机会**:
- 竞品均未将 NAS 作为核心归档目标
- YoYotta 专注 LTO，Silverstack 专注本地
- 国内市场 NAS 使用率高

**核心功能**:
- NAS 设备自动发现 (SMB/NFS/AFP)
- NAS 健康监控 (SMART、容量、RAID状态)
- 增量同步到 NAS
- NAS 快照管理
- 多 NAS 冗余备份
- NAS 归档报告

**技术方案**:
```typescript
interface NASConfig {
  protocol: 'smb' | 'nfs' | 'afp'
  host: string
  share: string
  credentials: {
    username: string
    password: string
  }
  healthCheck: {
    smartMonitoring: boolean
    capacityAlert: number  // 百分比
    raidStatus: boolean
  }
}
```

**预计工作量**: 6-8 周
**收益**: 填补市场空白，差异化竞争优势
